## Decisions & Notes

- **`'use client'` on printable documents**: although the document *pages* (`src/app/docs/...`) are server components, the document *components* (`customizable-invoice.tsx`, `generic-document.tsx`) are `'use client'` because they own the print button (`window.print()`). This matches the existing `documents.tsx` pattern.
- **Native `<select>` in designer/dialog**: Per the task brief, the encounter designer and encounter dialog use native `<select>` instead of Radix Select to avoid the well-known spacebar focus quirk in form-heavy dialogs.
- **API endpoints**: Several endpoints the new components depend on (`/api/erp/custom-fields/values`, `/api/erp/notes`, `/api/erp/attachments`, `/api/erp/clinical-encounter/[orderId]`, `/api/erp/encounter-template`, `/api/erp/document-templates`) do not exist yet. The components fail-fetch gracefully (empty state, no crash) so a future agent can add the API routes without touching these files.
- **`SearchableSelect` width**: The popover content uses `style={{ width: 'var(--radix-popover-trigger-width)' }}` to match the trigger width with a `minWidth: '14rem'` floor.
- **`computeCalculatedValue` `expression` case**: Implements a minimal safe-ish evaluator. `{field}` placeholders are replaced with the source value (or 0), then the result is run through a regex gate (`^[\d\s+\-*/.()]+$`) before being passed to `Function`. Non-conforming expressions return `''`.
